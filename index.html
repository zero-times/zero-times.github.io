Flutter 应用鸿蒙生态迁移

多端统一 · 地图与支付一致体验

主导基于 OpenHarmony Flutter SDK 的适配，封装地图与支付能力，完成鸿蒙系统首发。

  • 挑战： iOS/Android 的既有业务须在鸿蒙侧无缝承接。
  • 行动： 设计多端抽象层 + Platform Channel 通信，调用 HMS Core 能力。
  • 成果： 实现“一套代码，三端部署”，沉淀跨端迁移方法论。
高性能文件传输

NAS 上传稳定性提升 300%

构建基于 Isolate 的异步上传引擎，解决大文件上传卡顿、易失败的问题。

  • 挑战： 大文件传输占用主线程导致界面卡顿，失败率居高不下。
  • 行动： 设计切片与断点续传策略，配套自定义文件选择器与播放器。
  • 成果： 后台稳定上传 + 前台丝滑交互，形成可复用技术资产。
核心技术 SDK 化

为荣耀交付轻量直播 SDK

独立完成选型、架构与关键实现，把复杂直播业务萃取为可快速集成的 SDK。

  • 挑战： 多端直播能力需以低功耗、高稳定的 SDK 形态交付合作方。
  • 行动： 重构接口、加强容错、优化功耗并完成多端适配。
  • 成果： 通过严格测试并量产上线，实现技术能力产品化输出。
